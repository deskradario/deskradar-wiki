DeskRadar

Real-time local ADS-B aircraft tracking on a 64x64 LED matrix display.

DeskRadar is a hardware and software system that reads aircraft data from a local dump1090 ADS-B output and renders nearby aircraft tracks on a 64x64 RGB LED matrix. The Raspberry Pi runs the RTL-SDR receiver, display calculation software and support services. An Adafruit MatrixPortal S3 drives the HUB75 matrix over HTTP.

The main display shows each aircraft as a colored track. Position is mapped from latitude and longitude to matrix pixels, altitude is mapped to color, trails fade as the track ages, and duplicated pixels are collapsed so the highest aircraft wins when multiple points land on the same pixel. A 16x2 I2C LCD shows status messages and the current closest aircraft callsign/altitude.

Components

deskradar

Python application that polls ADS-B data, processes aircraft tracks through a multi-stage pipeline, and sends pixel draw commands to the matrix.

Language: Python 3.13
Runs on: Raspberry Pi (Linux)

deskradar-pi-services

Supporting services on the Pi: a FastAPI LCD display service, a Flask web configurator, and a boot-time mDNS resolver.

Language: Python
Managed by: systemd

deskradar-matrixportals3

CircuitPython firmware on an Adafruit MatrixPortal S3. Exposes an HTTP API and renders pixel commands to the physical 64×64 LED matrix.

Language: CircuitPython
Hardware: MatrixPortal S3 + HUB75 matrix

Data Flow Summary

  1. dump1090 receives ADS-B transmissions from aircraft and exposes them as JSON at :8080/data/aircraft.json.
  2. The deskradar app polls that endpoint every second by default, filters aircraft by altitude and computed latitude/longitude bounds, then runs each track through a 10-stage processing pipeline (plot → deduplicate → fill gaps → smooth → color by altitude → fade trails).
  3. The resulting pixel list is sent via HTTP POST to /draw on the MatrixPortal S3, which renders it onto the 64×64 LED matrix. Empty and unchanged frames use /empty and /no-change.
  4. Simultaneously, the closest aircraft callsign, altitude, and spinner are sent to the LCD service on the Pi, which displays them on a 16×2 character display.
  5. The web configurator allows changing the browser-exposed settings and restarts the main app when saved.

A note on AI

The deskradar and deskradar-pi-services Python codebases were written by me. AI was used as a tool for bug finding, writing specific functions, and code review — the core design and implementation is my own work.

The deskradar-matrixportals3 CircuitPython firmware was more heavily AI-assisted due to my limited experience with CircuitPython, but was reviewed and scrutinised by me throughout.

This documentation site and any READMEs were mostly written by AI for speed and convenience - (who enjoys writing this stuff?) and reviewed by me.
