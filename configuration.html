Configuration Reference

All fields in /etc/deskradar/config.json.

<<<<<<< Updated upstream

The configuration file is read at startup by the main deskradar app and validated by a Pydantic model. If the file is missing or contains invalid values for active fields, the app will exit with an error. Extra compatibility fields may be present in the JSON and are ignored by the main app. Changes take effect after restarting the service (sudo systemctl restart deskradar). The web configurator at http://deskradar.local:5000/config handles saving and restarting automatically.

Compatibility Fields

KeyTypeDefaultDescription
HARD_RESTARTboolfalse Present in the example config for compatibility. The current main app ignores this field.

Derived Fields (read-only)

These are computed from other config values and cannot be set manually. They are calculated at startup by the Pydantic model:

FieldDescription
THOLDSDictionary of north_lat, south_lat, east_lon, and west_lon computed from LAT, LON, and DISPLAY_RADIUS_NM using geopy's geodesic distance.
LAT_MIN / LAT_MAXSouthern and northern latitude bounds of the display area.
LON_MIN / LON_MAXWestern and eastern longitude bounds of the display area.
LAT_RANGENumPy linspace array of 64 latitude values evenly spanning LAT_MIN to LAT_MAX. Used for pixel mapping.
LON_RANGENumPy linspace array of 64 longitude values evenly spanning LON_MIN to LON_MAX.

Web Configurator

The web UI at http://deskradar.local:5000/config edits the fields rendered in its form. GET_URL, LCD_SEND_URL, MATRIX_HTTP_URL, HARD_RESTART, and derived fields are not shown; existing unshown keys are preserved when saving.

The UI handles type coercion automatically — integer fields stay integers, floats stay floats. The CLOSEST_COLOUR field is presented as a color picker and converted between hex and RGB dict format transparently.
