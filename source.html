Source

All DeskRadar repositories and what they do.

deskradar

The main Python application. Runs on the Raspberry Pi, polls ADS-B data from a local dump1090 instance every second by default, processes aircraft tracks through a 10-stage pipeline, and sends pixel draw commands to the MatrixPortal S3 over HTTP. It also sends the closest aircraft callsign and altitude to the LCD service.

Language: Python 3.13  |  Runs on: Raspberry Pi OS (Bookworm)

github.com/deskradario/deskradar

deskradar-pi-services

Supporting services that run alongside the main app on the Pi. Includes a FastAPI server that drives the 16×2 I2C LCD display, a Flask web configurator for changing settings and switching WiFi networks, and a one-shot boot service that resolves the MatrixPortal's mDNS address and writes it to the config file. The main app and configurator are ordered after bootstrap, but are not blocked if bootstrap fails.

Language: Python  |  Managed by: systemd

github.com/deskradario/deskradar-pi-services

deskradar-matrixportals3

CircuitPython firmware for the Adafruit MatrixPortal S3. The device reads wifi.txt, connects to WiFi, advertises itself via mDNS as deskradar-portal.local, and runs an HTTP server on port 80. It supports /ping, /draw, /empty, /no-change, and /add-constant. On receiving a /draw request it renders the pixel data to the 64×64 HUB75 LED matrix and redraws the fixed 2×2 grey center marker. A watchdog reboots the device if no request arrives within 60 seconds.

Language: CircuitPython  |  Hardware: Adafruit MatrixPortal S3 + 64×64 HUB75 LED matrix

github.com/deskradario/deskradar-matrixportals3

Firmware install

Copy the contents of src/ to the CIRCUITPY volume. The bundled installer is the macOS path variant:

cd deskradar-matrixportals3
./install.sh
